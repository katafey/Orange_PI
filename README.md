Краткий пересказ
Проверка: Сначала определяем имя диска (SD или eMMC) и адрес начала раздела.

Загрузка: Скачиваем файл working_boot_v1.4.bin по твоей прямой ссылке.

Прошивка: Записываем загрузчик через dd, пропуская первый сектор (MBR), чтобы не «убить» таблицу разделов.

Фиксация: Запрещаем системе обновлять загрузчик в будущем, чтобы ядра снова не «отвалились».

Пошаговая инструкция (Симуляция на чистой системе)
Шаг 1: Определение целевого устройства
Прежде чем шить, нужно понять, где живет система и не изменилась ли структура разделов.

Bash
sudo fdisk -l
На что смотрим:

Имя диска: Если загрузился с карты — это /dev/mmcblk0. Если уже перенес на eMMC — /dev/mmcblk2.

Сектор Start: В колонке Start первого раздела ты увидишь либо 8192, либо 32768.

Важно: Какой бы там ни был номер, наша команда dd с параметром seek=1 безопасна, так как она не трогает самый первый сектор (0-511 байт), где хранится таблица разделов.

Шаг 2: Скачивание фикса
Используем твою прямую ссылку на релиз:

Bash
wget https://github.com/katafey/Orange_PI/releases/download/Pi/working_boot_v1.4.bin

Шаг 3: Применение фикса (Хирургическая точность)
Записываем рабочий ATF/U-Boot. Мы используем bs=512 и seek=1, чтобы оставить родную таблицу разделов (MBR) нетронутой.

Если ты на SD-карте:

Bash
sudo dd if=working_boot_v1.4.bin of=/dev/mmcblk0 bs=512 seek=1 conv=notrunc status=progress
Если ты на eMMC:

Bash
sudo dd if=working_boot_v1.4.bin of=/dev/mmcblk2 bs=512 seek=1 conv=notrunc status=progress
Шаг 4: Блокировка официальных обновлений
Это критический шаг. Armbian при apt upgrade может попытаться вернуть свой «сломанный» загрузчик. Запрещаем это:

Bash
sudo apt-mark hold linux-u-boot-current-orangepi3-lts
Теперь при обновлениях системы загрузчик будет игнорироваться.

Шаг 5: Применение изменений
Bash
sync
sudo reboot
Проверка результата после перезагрузки
После старта проверяем, подхватило ли ядро все мощности процессора:

Количество ядер: nproc (должно быть 4).

Визуальный контроль: htop (должно быть 4 шкалы).

Лог загрузки: dmesg | grep -i psci.

Было: failed to boot CPU1 (-22).

Стало: Ошибки быть не должно, либо сообщения о корректном старте CPU.
